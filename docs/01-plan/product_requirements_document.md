# 문서 기반 검색 시스템 PRD (Product Requirements Document)

- **작성일**: 2026-04-08
- **버전**: v1.0
- **작성자**: PM Agent

---

## 1. 개요 및 비즈니스 맥락 (Background & Goals)

### Problem Statement (해결 과제)
내부 직원이 업무에 필요한 PDF 문서를 찾을 때, 수천 개의 파일을 직접 열어보거나 폴더를 일일이 탐색해야 하는 비효율이 존재한다.

### Business Objective (목표)
- 수천 개의 PDF 문서를 **3초 이내**에 키워드로 검색 가능
- 검색 결과에서 **문서 내 매칭 위치를 즉시 확인** (하이라이팅)
- 별도 교육 없이 누구나 바로 사용할 수 있는 단순한 UI

### Success Metric (성공 지표)
- 검색 응답 시간 **3초 이내** (p95 기준)
- 동시 접속자 **10명** 환경에서 성능 저하 없음
- 문서 추가 후 **5분 이내** 검색 반영

---

## 2. 시스템 구조 (Architecture)

### 채택 방식: A방식 (FSCrawler + Elasticsearch + Next.js)

```
[PDF 파일]
    /documents 폴더에 배치
          ↓ (자동 감지, 최대 5분)
[FSCrawler]
    PDF 텍스트 추출 + ES 인덱싱
          ↓
[Elasticsearch 8.x]
    - 역색인 구조로 빠른 검색
    - analysis-nori (한국어 형태소 분석)
    - 하이라이팅 내장
          ↓ (CORS 허용, 직접 호출)
[Next.js FE]
    단일 검색 화면
```

### 핵심 설계 원칙
- **BE 서버 없음**: FE가 ES REST API 직접 호출
- **Docker Compose**: 전체 환경 단일 명령으로 실행
- **볼륨 마운트**: ES 데이터 영속 보존 (Docker 재시작 시 재스캔 없음)

## 3. 기술 스택

| 영역 | 기술 | 버전 | 이유 |

## 4. 화면 구성 (단일 화면)

### 검색 메인 화면 (`/`)

**구성 요소:**
1. **검색 입력창** — 상단 중앙, 자동완성 없음 (속도 우선)
2. **검색 결과 목록**
   - 문서명
   - 매칭된 텍스트 스니펫 (하이라이팅 포함)
   - 페이지 번호
   - 파일 크기 / 최종 수정일
3. **결과 없음 상태** — 빈 결과 안내 메시지

### 비고
- 별도 문서 업로드 화면 없음 (폴더 직접 관리)
- 별도 관리자 화면 없음 (Kibana로 모니터링)

## 5. 핵심 기능 명세

### 5.1 검색 기능
| 항목 | 명세 |
|---|---|
| 검색 방식 | Elasticsearch BM25 (키워드 기반 관련도 순 정렬) |
| 언어 | 한국어 (nori 형태소), 영어 동시 지원 |
| 최소 입력 | 1자 이상 |
| 결과 정렬 | 관련도 점수 (score) 내림차순 |
| 페이지당 결과 | 10건 |

### 5.2 하이라이팅
| 항목 | 명세 |
|---|---|
| 방식 | ES 내장 highlight API |
| 표시 | 매칭 키워드 `<em>` 태그로 강조 |
| 스니펫 길이 | 매칭 전후 150자 |
| 스니펫 수 | 문서당 최대 3개 |

### 5.3 문서 인덱싱
| 항목 | 명세 |
|---|---|
| 지원 형식 | PDF |
| 감지 주기 | 5분 (설정 변경 가능) |
| 최초 실행 | Docker 기동 시 전체 폴더 자동 스캔 |
| 재시작 시 | 볼륨 유지로 변경 파일만 처리 |

## 6. 예외 처리 정책

| 상황 | 시스템 동작 |
|---|---|
| 검색 결과 없음 | "검색 결과가 없습니다" 메시지 표시 |
| ES 연결 실패 | "서버에 연결할 수 없습니다" 에러 메시지 |

| 텍스트 추출 불가 PDF (스캔본) | FSCrawler 스킵, 검색 대상에서 제외 |
